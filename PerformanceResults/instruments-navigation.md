## 第 6 步：如果还是看不到

### 检查以下几点：

#### 问题 A：主面板是空白的

**原因**：可能没有展开节点

**解决**：
1. 在主面板找到 "Root" 字样
2. 点击左侧的小三角形 ▶ 展开它
3. 或者双击 "Root"

#### 问题 B：看不到列标题

**原因**：可能在错误的视图中

**解决**：
1. 点击主面板顶部
2. 确保看到 "Running Time"、"Self"、"Symbol Name" 等列标题

#### 问题 C：显示的是 "Heaviest Stack"

**原因**：选中的是错误的视图

**解决**：
1. 点击顶部的 "☐ Heaviest Stack" 取消勾选
2. 点击 "☑ Call Tree" 勾选它

---

## 第 7 步：确认你看到 Call Tree

**你应该看到类似这样的内容**：

```
Root
  ├─ all (95%)
  │   ├─ <no symbol> (90%)
  │   │   ├─ mach_msg (10%)
  │   │   └─ ...
  │   └─ idle_wt... (5%)
  └─ other (5%)
```

**特征**：
- 有 "Root" 在最顶部
- 有 "├─" 和 "└─" 树状结构
- 有百分比数字，如 "(95%)"
- 可以上下滚动查看更多

## 🔧 实在找不到？用这个方法

### 方法 1：重置视图

在 Instruments 菜单中：
1. **View → Standard View**
2. 或者按 `Cmd + 1`

### 方法 2：点击左侧面板

1. **点击左侧的 "Time Profiler"**（不是三角形）
2. 等待右侧刷新
3. **点击展开的 "Run 1"**
4. **点击 "Call Tree"**（确保勾选）

### 方法 3：截图给我

如果还是找不到：
1. 按 `Cmd + Shift + 4` 截图
2. 选择整个 Instruments 窗口
3. 把截图给我看
4. 我告诉你应该点哪里

## 💡 替代方案：先做场景 2（更实用）

如果 Call Tree 实在找不到，**不要卡在这里**！

**建议**：
1. **先做场景 2 的操作**（20 次笔记创建/编辑）

   - CPU 峰值是多少？

3. **把观察结果告诉我**

**Activity Monitor 更直观**，而且能发现大部分性能问题！

Call Tree 主要用于：
- 找到具体哪个函数慢
- 分析调用栈

但首先，我们需要确认**是否有性能问题**，Activity Monitor 就够了！
