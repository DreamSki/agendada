## 🔍 查看关键数据

### 步骤 1：确保在 Call Tree 视图

1. 在左侧面板，点击 **"Run 1: 30.02s"**
2. 确保勾选了 **"☑ Call Tree"**

### 步骤 2：查看应用总体占用

在 Call Tree 中：

1. **找到 "Agendada" 节点**
   - 展开直到看到类似：
   ```
   Root
     ├─ (0%) all
     └─ (95%) Agendada        ◀─ 这里的 95% 是什么？
       ├─ (90%) main
       └─ (5%) 其他
   ```

2. **查看百分比**：
   - **Weight %**：该函数（包含子调用）占总时间的比例
   - **Self %**：该函数自身的执行时间占比

### 步骤 3：查看 CPU 相关信息

虽然 Time Profiler 不直接显示 CPU%，但你可以：

1. **查看采样频率**：
   - 在底部 "Display Settings"
   - 查看 "Sample Rate"：通常是 1 ms（1000 Hz）
   - 如果 Agendada 占 95%，说明 CPU 使用率很高

2. **查看运行时间分布**：
   ```
   Run 1: 30.02s
   - Agendada: 28.5s (95%)    ← 说明几乎一直在运行
   - Idle: 1.5s (5%)         ← 空闲时间很少
   ```

## 💡 关键指标解释

### 你应该看到的（基线空闲）

对于**空闲 30 秒**的基线测试，预期结果：

```
Agendada:
- Weight %: 应该 < 20%（说明大部分时间空闲）
- 主要函数应该显示 "idle" 或 "runloop" 相关

如果看到：
- Agendada 占 > 80%  → ❌ 应用持续工作，可能有问题
- 看到 snapshot() → ❌ 在后台执行保存操作
- 看到 filteredNotes() → ⚠️ 可能在刷新视图
```

### 具体检查函数

在 Call Tree 顶部的搜索框（🔍）中搜索：

1. **`snapshot`**
   - 如果出现 → 说明在执行保存
   - 查看 "Self" 时间（应该很小）

2. **`publishChange`**
   - 如果出现 → 说明状态在变化
   - 查看调用次数（在 "Count" 列）

3. **`filteredNotes`**
   - 如果出现 → 说明视图在刷新
   - 基线应该很少或没有
